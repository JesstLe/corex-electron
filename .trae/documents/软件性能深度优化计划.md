我将制定一个详细的计划，以实现您提出的性能优化和功能增强需求。

### 1. 性能优化 (Smart Refresh)
*   **目标**：彻底解决 CPU 占用问题，实现“零打扰”。
*   **行动**：
    *   **取消自动刷新**：移除 `app.rs` 中 1000ms 的定时数据刷新逻辑，改为仅在用户点击“刷新”按钮或特定操作时触发。
    *   **智能 UI 渲染**：引入 `winit` 的 `Focused` 事件监听。
        *   当窗口**有焦点**时：保持 200ms (5 FPS) 的低频重绘。
        *   当窗口**失去焦点/最小化**时：停止主动重绘 (`request_repaint`)，仅响应系统事件（如托盘点击），将 CPU 占用降至 0%。

### 2. 记忆功能 (Process Whitelist)
*   **目标**：免去重复配置的烦恼，实现“一次设置，永久生效”。
*   **行动**：
    *   **配置文件**：在 `config.json` 或新建 `profiles.json` 中存储进程策略白名单。
        *   结构：`HashMap<ProcessName, Profile>`。
    *   **自动应用**：
        *   程序启动时，加载白名单。
        *   用户点击“应用”时，自动将当前配置保存到白名单。
        *   新增“自动应用”开关：开启后，后台线程会定期扫描进程列表，发现白名单中的进程就自动应用策略（类似于之前的 Watchdog，但更精准）。
        *   **快捷添加**：UI 增加“保存到预设”按钮，或默认行为即保存。

### 3. 电源计划管理 (Power Plan Manager)
*   **目标**：一站式管理 Windows 电源计划。
*   **行动**：
    *   **后端 (`src/core/power.rs`)**:
        *   封装 `powercfg` 命令：
            *   `list`: 获取所有可用计划（解析 GUID 和名称）。
            *   `set_active`: 切换计划。
            *   `import`: 导入 `.pow` 文件。
            *   `delete`: 删除计划（需过滤掉系统默认的平衡/高性能/节能，防止误删）。
    *   **前端 UI (`src/ui/power.rs`)**:
        *   新增“电源管理”面板。
        *   **列表显示**：列出所有计划，高亮当前活动计划。
        *   **切换**：点击即切换。
        *   **导入**：支持文件拖拽区域，识别 `.pow` 文件并导入。
        *   **删除**：允许删除非系统默认计划。
