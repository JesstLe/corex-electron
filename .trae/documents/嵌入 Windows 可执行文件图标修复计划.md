我将修复可执行文件（`.exe`）图标丢失的问题。

### 问题原因
Rust 默认构建生成的可执行文件没有嵌入图标资源。虽然代码中加载了窗口图标，但这只影响运行时窗口左上角和任务栏的显示，不影响 Windows 资源管理器中 `.exe` 文件本身的图标。

### 修复方案
1.  **添加构建依赖**：引入 `winres` 库，用于在构建过程中编译 Windows 资源文件。
2.  **创建构建脚本**：新建 `build.rs`，编写逻辑将 `icon.ico` 嵌入到最终的可执行文件中。
3.  **重新构建**：再次执行 `cargo build --release`。

1.  修改 `Cargo.toml` 添加 `[build-dependencies]`。
